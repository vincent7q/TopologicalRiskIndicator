# TDA-RiskPulse：概念與原理說明（繁體中文）

> 本文以非技術與技術兩個層面，說明本專案的核心概念、目標，以及「為什麼拓撲學能在危機發生**之前**偵測到風險」，並解釋**拓撲熵（Topological Entropy）**的運作邏輯。
>
> 對應的正式文件：產品需求見 `PRD.md`、數學與實作規格見 `SPEC.md`、資產清單見 `nasdaq_tda_portfolio.md`。

---

## 1. 專案目標

**TDA-RiskPulse** 是一個**領先型（leading）市場風險指標**。

傳統的風險指標（VIX 恐慌指數、滾動相關係數、波動率）幾乎都是**落後指標（lagging）**——它們要等到價格已經劇烈波動之後，數值才會上升。等到 VIX 飆高時，崩盤往往已經開始，對風險控管者來說為時已晚。

本專案的目標：在**價格尚未崩跌之前**，從市場的「結構」中找出即將失穩的徵兆，提供一個**提前預警**的訊號。

核心假設（thesis）：

> 市場在大跌之前，會先進入一種「結構脆化（structural embrittlement）」的狀態。這種脆化會以一種**幾何 / 拓撲**的型態顯現出來，而這個型態的變化，**早於**價格的崩潰。

---

## 2. 核心直覺：把市場看成一團「點雲」

關鍵的觀念轉換是：**不要只看單一資產的價格，而是看「所有資產之間的結構關係」。**

作法：

1. 取 N 檔資產（本專案約 48–65 檔：Nasdaq 權值股 + 跨資產的流動性/宏觀指標，如債券、黃金、原油、匯率等）。
2. 把每檔資產的**日對數報酬率（log return）**當成一個維度。
3. 於是「某一段時間內的市場狀態」就變成一個**高維空間中的點雲（point cloud）**——每一檔資產是一個點，點與點之間的「距離」由它們報酬率的相關性決定。

這團點雲的**形狀**，就是市場的「結構」。

### 為什麼形狀會洩漏危機？

- **正常時期（健康市場）**：各資產各走各的路，相關性低而分散。點雲是**散開、鬆散、沒有特別結構**的——拓撲上很「無聊」（只是一堆分離的小群集）。
- **危機醞釀期（脆化）**：恐慌與去槓桿讓資金同進同出，資產之間的相關性**普遍升高但又不均勻地糾纏在一起**。原本分散的點被拉攏、重組，點雲開始出現**環狀結構（loops / holes）**——也就是拓撲上的「洞」。

這個「洞」的出現，代表市場的相依結構從「多元分散」塌縮成「高度耦合但脆弱」的狀態。**這種結構重組，通常發生在價格崩潰之前**——這正是領先訊號的來源。

> 這個想法源自金融拓撲資料分析的奠基研究（Gidea & Katz, 2018），他們發現在 2000 網路泡沫與 2008 金融海嘯**之前**，持續同調（persistent homology）的特徵強度會明顯放大。

## 3. 拓撲學工具：持續同調（Persistent Homology）

我們需要一個數學工具，能客觀地「量測點雲的形狀」，特別是偵測**環狀的洞**。這個工具就是**持續同調**。

幾個關鍵名詞：

- **$H_0$**：連通分量（clusters，幾群點黏在一起）。
- **$H_1$**：一維的洞 / 環（loops，點雲圍成一個圈）。**這是本專案最關注的特徵**——危機前的結構脆化，主要以 $H_1$ 環的形式出現。

### 運作流程（逐步）

#### 步驟 1：報酬率（讓資料平穩）
把原始價格 $P_{i,t}$ 轉成日對數報酬率：

$$R_{i,t} = \ln\!\left(\frac{P_{i,t}}{P_{i,t-1}}\right)$$

#### 步驟 2：相關性 → 距離矩陣
對每一個滾動時間視窗（本專案 $\tau = 40$ 個交易日），先算出資產之間的皮爾森相關係數矩陣 $\rho$，再把「相關性」轉換成「距離」：

$$D_{i,j} = \sqrt{2\,(1 - \rho_{i,j})}$$

這個轉換的直覺很重要：

| 相關係數 $\rho$ | 距離 $D$ | 意義 |
|:---:|:---:|:---|
| $\rho = 1$（完全同向） | $D = 0$ | 兩資產行為一致 → 在點雲中**重疊** |
| $\rho = 0$（無關） | $D = \sqrt{2} \approx 1.41$ | 中等距離 |
| $\rho = -1$（完全反向） | $D = 2$ | 行為相反 → 在點雲中**最遠** |

於是「相關性高」就等於「點靠得近」，市場的相依結構被忠實地翻譯成幾何上的距離。

#### 步驟 3：Vietoris–Rips 過濾，找出洞
想像把每個點慢慢「長大成球」，半徑 $\epsilon$ 從 0 開始增加。當兩球相交就連一條線、三球相交就補一個面……隨著 $\epsilon$ 變大，點雲逐漸被填滿。

在這個過程中：
- 某些**環（$H_1$ 洞）**會在某個半徑「**誕生（birth）** $b_k$」，又在更大的半徑被填補而「**死亡（death）** $d_k$」。
- 一個洞活得越久（$d_k - b_k$ 越大），代表它是越「**真實、穩固**」的結構，而不是隨機雜訊。

這個「壽命」就是**持續性（persistence）**：

$$L_k = d_k - b_k$$

把所有 $H_1$ 洞的（誕生、死亡）畫成一張圖，就是**持續圖（persistence diagram）**。

## 4. 拓撲熵（Topological Entropy）的邏輯

持續圖裡可能有很多個洞，每個洞壽命不一。我們需要**把整張圖濃縮成一個數字**，方便畫成每日的時間序列——這就是**拓撲熵**。

它借用了資訊理論「熵」的概念來描述**結構的多樣性 / 集中度**。

### 公式（本專案 SPEC §2 的定義）

設某個視窗在 $H_1$ 維度有 $M$ 個洞，第 $k$ 個洞壽命為 $L_k$，總壽命 $S = \sum_{k=1}^{M} L_k$。

1. 把每個洞的壽命正規化成「機率權重」：

$$p_k = \frac{L_k}{S}$$

2. 計算**正規化拓撲熵**：

$$E = -\frac{1}{\ln M} \sum_{k=1}^{M} p_k \ln(p_k)$$

前面的 $\dfrac{1}{\ln M}$ 是正規化因子，讓 $E$ 永遠落在 **$[0, 1]$** 之間，方便跨時間比較。

> **實作重點**：本專案**親手實作這條公式**，而**不是**用 giotto-tda 內建的 `PersistenceEntropy(normalize=True)`。因為 giotto 的正規化是除以 $\ln(\sum L_k)$（總壽命），而非規格所要求的 $\ln M$（洞的數量）；前者在真實資料上會產生無界的、上千的數值。詳見 `tda_engine.normalized_h1_entropy`。
>
> **邊界情況**：當一個視窗的 $H_1$ 洞少於 2 個（$M < 2$）時，$\dfrac{1}{\ln M}$ 沒有定義（$\ln 1 = 0$），此時直接令 $E = 0.0$。

### 這個數字代表什麼？

$E$ 衡量的是：**整個洞結構的「持續性」是平均分散，還是集中在少數幾個洞上。**

- **高熵（$E \to 1$）**：許多洞的壽命**差不多長**——結構是「均勻、瀰漫」的。
- **低熵（$E \to 0$）**：少數幾個**主導性的洞**壟斷了大部分的結構持續性。
- **$E = 0$**：視窗中幾乎沒有環狀結構（拓撲上瑣碎，屬於正常的健康散開狀態，或為上述邊界情況）。

當市場從「多元分散」走向「結構脆化」，$H_1$ 洞的型態（數量與壽命分布）會發生顯著變化，拓撲熵就會偏離它平時的水位。本專案以 **$\mu + 2\sigma$**（歷史平均加兩個標準差）作為「危機觸發門檻」，把偏離正常的時點標示為**壓力區（stress zone）**。

## 6. 重要的誠實聲明

- **這是研究型指標，不是投資建議。** 在規格所定義的 $\mu + 2\sigma$ 規則下，訊號相當**稀疏**（20 年間僅數次突破），而且經驗上這些高熵尖峰**較傾向與**壓力期（2011、2015、2020）**同時發生**，而非乾淨地**領先**。要把拓撲熵真正當成穩定的領先指標，仍需進一步調校（例如改用滾動門檻、縮短時間跨度、或同時搭配持續性範數 persistence-norm 訊號）。
- **資產數量取決於時間跨度。** 引擎要求所有視窗使用**固定**的資產集合，因此在選定區間內缺漏超過 5% 的資產會被剔除。在預設約 20 年的視窗下，65 檔中僅約 48 檔存活（TSLA、META、AVGO、CRWD、IBIT 等較晚上市者會被排除）。**縮短日期區間**可保留更多權值股。
- **2015 年以前的訊號較為近似**——Nasdaq 成分股在二十年間會變動，點雲在較早的時段只能視為方向性參考。

## 7. 一句話總結

> 把市場想成一團會變形的「點雲」：太平時它鬆散無形，崩盤前它會悄悄糾結成「環」。**持續同調**負責偵測這些環，**拓撲熵**把環的結構濃縮成一個 0 到 1 的每日數字——我們監看這個數字何時異常偏離常態，藉此在價格崩潰之前，捕捉市場「結構脆化」的領先徵兆。
