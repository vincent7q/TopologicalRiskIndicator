# Technical Specification Document (SPEC)

## 1. System Architecture & Tech Stack
The application is a Python-based pipeline designed for mathematical batch processing.

* **Language:** Python 3.10+
* **Data Ingestion:** `yfinance` (historical market data API)
* **Data Wrangling:** `numpy`, `pandas`, `scipy`
* **TDA Framework:** `giotto-tda` (preferred for its native Scikit-Learn pipeline compliance and speed) or `ripser` + `persim` as a fallback.
* **Visualization:** `matplotlib` or `plotly`

## 2. Mathematical Framework & Formulations

### Step 1: Feature Stationarity
For a portfolio of $N$ stocks ($N=50$), raw prices $P_{i,t}$ are converted to daily log returns:
$$R_{i,t} = \ln\left(\frac{P_{i,t}}{P_{i,t-1}}\right)$$

### Step 3: Topological Entropy
Given a persistence diagram with $M$ persistence pairs in dimension 1 ($H_1$), where each bar $k$ has a birth time $b_k$ and death time $d_k$, the persistence length is $L_k = d_k - b_k$. 
The total length is $S = \sum_{k=1}^M L_k$. The probability weight is $p_k = \frac{L_k}{S}$.
The Normalized Topological Entropy $E$ is calculated as:
$$E = -\frac{1}{\ln M} \sum_{k=1}^M p_k \ln(p_k)$$

## 3. Implementation Blueprint & Module Specification

Your code generation must be split into three core script modules:

### Module 1: `data_fetcher.py`
* **Input:** Ticker list (`List[str]`), benchmark string, `start_date`, `end_date`.
* **Logic:** * Download Adjusted Close data via `yfinance`.
    * Calculate daily log returns.
    * Perform an active drop of any row containing `NaN` globally (forward-fill if minor gap, drop if systemic trading halt) to ensure strict matrix alignment.
* **Output:** Transformed multi-column `pandas.DataFrame` of stationary log returns.

### Module 2: `tda_engine.py`
* **Input:** Multi-column log return dataframe, `window_size` (default: 40), `step_size` (default: 1).
* **Logic:**
    * Iterate through rows via a sliding window.
    * For each window slice, calculate the Pearson Correlation matrix, then convert to a $N \times N$ Distance Matrix.
    * Initialize `gtda.homology.VietorisRipsPersistence(metric="precomputed", homology_dimensions=[0, 1])`.
    * Feed the distance matrix to extract the persistence diagram.
    * Initialize `gtda.diagrams.PersistenceEntropy(normalize=True)` to extract entropy metrics for $H_1$.
* **Output:** A time-series array/dataframe containing `[Date, H1_Entropy]`.

### Module 3: `main.py` & Visualizer
* **Logic:**
    * Coordinate modules 1 and 2.
    * Calculate a static anomaly boundary threshold: $\mu_{entropy} + 2\sigma_{entropy}$ over historical benchmarks.
    * Plot using `matplotlib`: Panel A (Benchmark index index line chart), Panel B (Topological Entropy line plot overlayed with the threshold).

## 4. Edge Cases & Error Handling to Guard Against
* **Low Feature Count inside Persistence Diagram:** If a rolling period exhibits near-perfect linear alignment, $H_1$ diagram may return 0 or 1 point, making `normalize=True` crash on division by zero inside `1/ln(M)`. Implement a fallback catch: if count of pairs $< 2$, return `Entropy = 0.0`.
* **Data Incommensurability:** If historical stock tracking includes bankrupt or newly listed entities inside the long multi-year horizon, the code must dynamically filter out any asset that contains $> 5\%$ missing rows in the current slice, replacing it with the next largest liquid asset.
